# Field-level data contracts for mutation tools

## Context

Wheeler mutation tools (`add_finding`, `add_dataset`, etc.) accept args from the LLM via MCP and store them in Neo4j + JSON + synthesis (triple-write). There is no input validation layer: confidence can be 5.0, paths can be relative or wrong, status fields accept any string. Only `tier` (in `set_tier`) and `relationship` (in `link_nodes`) are currently validated.

The goal: add a per-field validation/normalization layer that runs in `execute_tool()` **before** dispatch, so all three write layers receive clean data. Structured errors tell the LLM exactly what's wrong so it can self-correct.

## Current state: what is and isn't validated

**Validated today:**
- Tier: "reference" or "generated" only (in `set_tier`, mutations.py:447)
- Relationship type: against `ALLOWED_RELATIONSHIPS` whitelist with alias mapping (in `link_nodes`, mutations.py:354)
- Node ID prefix: must map to `PREFIX_TO_LABEL` (in `link_nodes`/`set_tier`/`delete_node`)
- Path: resolved to absolute, warns if missing on disk (ad-hoc, just added in `execute_tool`)

**Not validated (accepted as-is):**
- Confidence: no range check (0.0-1.0 documented but not enforced)
- Priority: no range check (1-10 documented but not enforced)
- Year: no range check
- Status (hypothesis/document/execution): no enum check
- Description/statement/question: can be empty even when required
- Path format (only existence, and only after the ad-hoc fix)

Adversarial tests in `tests/test_adversarial.py` confirm all of these gaps.

## Design

### Architecture placement

```
_field_specs.py   <- pathlib only (new leaf module, zero internal imports)
  ^
__init__.py       <- imports validate_and_normalize, calls in execute_tool()
  ^
mutations.py      <- unchanged, receives pre-validated args
```

### Public interface

```python
# wheeler/tools/graph_tools/_field_specs.py

def validate_and_normalize(tool_name: str, args: dict) -> tuple[dict, dict]:
    """Validate and normalize mutation tool arguments in-place.

    Returns (errors, warnings) where:
    - errors: {field_name: {"value": original, "error": message}}
    - warnings: {field_name: message}

    Only runs for known mutation tools. Returns ({}, {}) for unknown tools.
    Collects ALL errors before returning (no short-circuit).
    """
```

### Field specs

| Field | Normalize | Error if | Warn if |
|-------|-----------|----------|---------|
| `path` | `Path.resolve()` to absolute | - | not on disk |
| `confidence` | `float()` coerce | not in [0.0, 1.0] | - |
| `tier` | lowercase strip | not in {generated, reference} | - |
| `priority` | `int()` coerce | not in [1, 10] | - |
| `year` | `int()` coerce | not int | value is 0 |
| `status` | - | not in tool-specific enum | - |

### Required fields per tool

Error if missing from args or empty string:

- `add_finding`: description, confidence
- `add_hypothesis`: statement
- `add_question`: question
- `add_dataset`: path, type, description
- `add_paper`: title
- `add_document`: title, path
- `add_note`: content
- `add_script`: path, language
- `add_execution`: kind, description

### Status enums per tool

- `add_hypothesis`: open, supported, rejected
- `add_document`: draft, revision, final
- `add_execution`: completed, failed, running

### Excluded from validation

- `add_ledger`: system-generated by the citation validation pipeline, not LLM-authored
- Provenance fields: `execution_kind`, `used_entities`, `execution_description`, `agent_id`, `session_id`, `started_at`, `ended_at` are handled by `_complete_provenance` with their own edge-case behavior

### Design decisions

1. **Collect ALL errors** before returning, so LLM can fix everything in one retry
2. **Missing optional fields do NOT error** (handler defaults apply: status defaults to "open", priority to 5, etc.)
3. **Type coercion**: confidence/priority as strings get coerced via `float()`/`int()` before range check (LLMs sometimes send "0.5" instead of 0.5)
4. **Path normalization** happens in-place on `args` so downstream handlers receive absolute paths
5. **Errors block writes**: validation failure returns immediately, no graph/JSON/synthesis write occurs
6. **Warnings don't block**: path not existing is a warning attached to the success result

## Integration point: `execute_tool()`

Replace the ad-hoc path normalization in `wheeler/tools/graph_tools/__init__.py` with:

```python
from ._field_specs import validate_and_normalize

# In execute_tool(), before dispatch:
warnings = {}
if tool_name in _MUTATION_TOOLS:
    errors, warnings = validate_and_normalize(tool_name, args)
    if errors:
        return json.dumps({"error": "validation_failed", "fields": errors})

result = await handler(backend, args)

# After handler returns, attach warnings:
if warnings:
    try:
        parsed_result = json.loads(result)
        parsed_result["warnings"] = warnings
        result = json.dumps(parsed_result)
    except Exception:
        pass
```

## Error format (what the LLM sees)

Validation failure:
```json
{
  "error": "validation_failed",
  "fields": {
    "confidence": {"value": 5.0, "error": "must be 0.0-1.0"},
    "tier": {"value": "maybe", "error": "must be 'generated' or 'reference'"}
  }
}
```

Success with warnings:
```json
{
  "node_id": "F-3a2b",
  "label": "Finding",
  "status": "created",
  "warnings": {"path": "does not exist on disk: /Users/.../missing.csv"}
}
```
